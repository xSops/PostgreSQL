Pré-requis :
- avoir Docker sur sa machine hôte

Utilisation :
- Récupérer le projet depuis Git dans le répertoire de son choix
- Lancer Docker et se rendre dans ce répertoire
- La base de donnée par défaut s'appelle TEST et s'initialise automatiquement grace au répertoire : docker/sql-init/dumpfile-database.sql !! Vous pouvez ajouter autant de base de donnée que vous le souhaitez en déposant simplement au sein de ce répertoire vos fichiers .SQL
- Lancer la commande : docker-compose up -d (lance le container avec les scripts + BDD en mode Daemon)

Pour sauvegarder :
- docker-compose exec db bash docker-script-db/dump.sh
- Vous trouverez votre fichier SQL sur votre machine hôte : docker/sql-init/dumpfile-database.sql
- Pour changer la base de donnée : ouvrir le script bash : docker/script-db/dump.sh
	Remplacer à la ligne 3 la DB "test" par la DB voulue, ou par "--all-databases" pour toutes les bases.

Pour importer :
- Grâce à l'environnement Docker et le container MYSQL avec un point de montage l'import d'une BDD se fait automatiquement
- Retourner dans le répertoire d'initialisation : docker/sql-init
- Y déposer vos fichiers à exporter au format .SQL (ou ne rien toucher si vous souhaitez exporter vos précédentes sauvegardes)
- Lancer la commande : docker-compose down -v
- Lancer à nouveau la commande : docker-compose up -d
- DONE : Les bases ont été automatiquement importées.
